Images

There are four main image formats in use on the web that are all designed to do different things:

  • JPG
  • GIF
  • PNG
  • SVG


JPG Images

JPG images are designed for handling large color palettes without dramatically increasing file size.

PROs:

  • Great for images with gradients in them
  • Great for photos
CONs:
  • Don't allow for transparent pixels


GIF Images

GIF are for simple animations. However, they're very limited in terms of color palette.

PROs:

  • Great to show animations or memes
CONs:
  • Can't have semi-opaque pixels
  • Difficult to contain high levels of detail on transparent background


PNG Images

PNGs are great for anything that's not a photo or animated. They work well with opacity and don't have color palette limitations.

PROs:

  • Great for icons, technical diagrams, and logos
CONs:
  • PNG file of same quality would be bigger than equivalent JPG file


SVG Images

SVG is a vector-based graphics format, unlike the previous image types. Vector-based graphics can scale up or down to any dimension without loss of quality. amazing

PROs:

  • Great for responsive design
  • Good for same use cases as PNGs
CONs:
  • For consistency across browsers, you must convert any text fields to outlines using an image editor
  • Images that contain a lot of text can have impact on file size



Image Dimensions

By default, the <img/> element uses the inherit dimensions of it image file. The images on this page all have 150x150 pixels, while the SVG uses 75x75 pixels.

Pixel-based image formats need to be twice as big as you want them to appear on retina displays. To modify these images down to 75x75, you can use <img/> element's width attribute.
